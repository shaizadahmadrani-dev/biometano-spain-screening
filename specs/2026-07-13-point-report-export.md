# Detailed point report export

## Context

The Streamlit app already exports a comparison CSV and a one-page PDF snapshot
for the selected cell. That snapshot is useful as a compact record, but it does
not expose the technical indicators, evidence status, critical gates and source
limitations needed to review a point with a non-technical project stakeholder.

## Goal

Allow the user to download a professional, self-contained PDF report for the
currently selected 5 km or 1 km cell.

## Product contract

1. The report identifies the cell, scale, coordinates, province and autonomous
   community.
2. It separates screening priority, evidence completeness and prefactibility.
3. It explains strengths, weaknesses, review flags and physical exclusions.
4. It presents the main supply, infrastructure, land, environmental, terrain
   and hydrology indicators with their evidence level and interpretation.
5. It lists every critical prefactibility gate as pending, failed or supported
   by supplied evidence; missing data must never be rendered as compliance.
6. It includes source vintages, method version, snapshot date and limitations.
7. It links the coordinates to OpenStreetMap and Google Maps for parcel-level
   follow-up, without embedding or claiming cadastral validation.

## Safety language

- The score is a relative priority for acquiring evidence, not a probability.
- The report is not a declaration of viability, permission, land ownership,
  grid capacity or project profitability.
- Distance to gas, electricity and roads is a geometric proxy only.
- Nitrate-vulnerable zones activate digestate review; they are not a universal
  siting veto.
- A negative or empty screening layer is not proof that a constraint is absent.

## Architecture

- Keep the existing `pdf_bytes` one-page snapshot API backwards compatible.
- Add a separate pure report module with:
  - a deterministic structured-data builder suitable for semantic tests;
  - a PDF renderer with no Streamlit dependency;
  - a filesystem export wrapper and safe filename helper.
- The Streamlit layer only gathers the selected row and manifest, then exposes
  the generated bytes through a download button.

## Validation

- Unit-test missing-value semantics, critical-gate classification, technical
  proxy wording, map links and filename sanitisation.
- Assert that the generated artifact is a multi-page PDF.
- Preserve all existing snapshot PDF tests.
- Run the focused export tests and the complete app suite without a build.

## Non-goals

- Do not produce a cadastral, environmental, engineering or investment dossier.
- Do not fetch live registry or map data while generating the report.
- Do not alter scores, model artifacts, map layers or model promotion status.

## Closeout

Status: complete. The app keeps the original one-page summary and adds a
four-section detailed PDF for the selected cell. Semantic tests cover missing
data, gate states, proxy wording, links and filenames. Both public 5 km and 1 km
snapshots generated four-page reports successfully; the complete app suite and
the independent model/artifact suite passed without changing the deployed
model.
